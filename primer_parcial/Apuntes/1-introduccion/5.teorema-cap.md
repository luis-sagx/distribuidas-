# Teorema CAP

- propuesto por Eric Brewer en 2000
- define el comportamiento de los sistemas distribuidos en términos de tres propiedades: Consistencia, Disponibilidad y Tolerancia a Particiones.
- un sistema distribuido es un grupo de nodos que se comunican entre sí a través de una red y trabajan juntos para lograr un objetivo común.

- para una operacion de escritura seguida de una lectura, el sistema solo puede garantizar dos de tres atributos simultaneamente, debiendo sacrificar el tercero

## Consistencia

- todos los nodos ven la misma información al mismo tiempo. Si un nodo actualiza un dato, todos los demás nodos deben ver esa actualización inmediatamente.

## Disponibilidad - Availability

- Si un nodo se encuentra abajo o no responde, el sistema sigue funcionando y puede responder a las peticiones de los usuarios. No se garantiza que los datos estén actualizados, pero el sistema sigue siendo accesible.

## Tolerancia a Particiones - Partition Tolerance

Un sistema debe seguir funcionando aun cuando los nodos no se puedan comunicar entre si (temporal o permanentemente). Esto puede ocurrir debido a fallas de red, cortes de energía o problemas de hardware.

## Ejemplo

- Debemos tener en cuenta que en la computacion distribuida la red nunca es confiable las caidas de red ocurren siempre y no las podemos controlar, por lo tanto, nos queda escoger entre Consistencia y Disponibilidad, pero no podemos tener ambos al mismo tiempo.

#### CP

- Redis, MongoDB, HBase, Zookeeper
